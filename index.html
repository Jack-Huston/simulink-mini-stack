Overview

Built to demonstrate a realistic GN&C workflow: plant dynamics + sensors + estimation/signal processing + controller + verification.

Plant + Actuation

  • Small-sat attitude dynamics with 3D visualization workflow
  • Reaction wheel actuation (limits/saturation as applicable)
  • Disturbance injection for robustness testing

Sensors + Noise

  • IMU model (gyro/accel behavior as implemented)
  • Star sensor measurement model + update rate effects
  • Configurable measurement noise and bias terms (where applicable)

Onboard Processing

  • Signal conditioning blocks feeding the controller
  • Interfaces structured like a flight-style pipeline
  • Scenario parameterization for repeatable runs

What this demonstrates (for recruiters)

  • Model-based development in Simulink (not just “I used it once”)
  • Control loop performance reasoning with noise + disturbances
  • A verification mindset: repeatable scenarios, metrics, and robustness sweeps

Validation approach

The point isn’t “it runs.” The point is: the outputs are defensible, repeatable, and stress-tested.

Tracking performance
Attitude error vs time; settling time; steady-state error.
Control effort
Wheel torque/speed limits; saturation time; command smoothness.
Robustness
Noise sweeps + disturbance cases; Monte Carlo pass/fail criteria.
Repeatability
Scenario presets; fixed seeds (where used); reproducible results.
What will be published here
  • “How to run” guide + scenario presets
  • Plots for baseline tracking + noise/disturbance sweeps
  • Clear assumptions + limitations (so the model doesn’t over-claim)
